# Contributing Guidelines

Thank you for your interest in contributing to this project!
We welcome contributions from anyone.

---

## How to Contribute

1. Fork the repository  
2. Create a new branch: `feature/{prefix}-your-feature-name`  
3. Make your changes following the [Code Style](#code-style)  
4. Make sure to test on real devices, IOS and Android. If not possible please clarify what you did and didn't test.
5. Submit a pull request (PR) with a clear description

---

## Code Style

- Use idiomatic Angular and TypeScript  
- Run formatters and linters before committing:  
  `npm run format && npm run lint`
- Run tests before committing:  
  `npm run test`

---

## Commits
Make sure to explain in the commit message what changed in the commit and why it changed.
ALways prefer smaller multiple commits over bigger commits. If commits are to big we might decline the PR.
